## Expanded Page Notes

This section shows each handwritten page at a larger display size and then explains only the ideas written on that page. Use this when the small day-file thumbnails are not enough for revision.

### `till46.pdf`

#### till46 page 001

<img src="images/HandWrittenNotes/till46/page-001.jpg" width="720">

Explanation: This page starts the 8085 from inside the CPU. It writes the ALU, accumulator, temporary register, flag register, and control unit as the main working blocks. Read it as: data goes into the ALU, the result is collected in the accumulator, and the flag register records the status of that result through `S`, `Z`, `AC`, `P`, and `CY`.

#### till46 page 002

<img src="images/HandWrittenNotes/till46/page-002.jpg" width="720">

Explanation: This page expands the internal register set. It separates the general-purpose registers `B`, `C`, `D`, `E`, `H`, `L`, the accumulator, temporary registers, `PC`, `SP`, instruction register, decoder, and timing/control unit. The important idea is that the 8085 has 8-bit working registers, but uses 16-bit register pairs and 16-bit addresses for memory operations.

#### till46 page 003

<img src="images/HandWrittenNotes/till46/page-003.jpg" width="720">

Explanation: This page is the quick feature sheet of the 8085. It records the 40-pin processor, 8-bit data bus, 16-bit address bus, `2^16 = 64 KB` addressable memory, tri-state bus behavior, serial I/O, interrupts, and software compatibility idea. Use it to remember what makes the 8085 an 8-bit microprocessor.

#### till46 page 004

<img src="images/HandWrittenNotes/till46/page-004.jpg" width="720">

Explanation: This page explains the pin diagram by groups. It notes the address pins, multiplexed `AD0-AD7` pins, clock pins, reset, `READY`, `HOLD/HLDA`, interrupts, serial pins, and control/status pins. The purpose is to stop memorizing the 40 pins randomly and instead group them by function.

#### till46 page 005

<img src="images/HandWrittenNotes/till46/page-005.jpg" width="720">

Explanation: This page focuses on status and control signals. `IO/M`, `S1`, and `S0` identify the type of machine cycle, while `/RD`, `/WR`, and `/INTA` perform the read, write, or interrupt-acknowledge action. The handwritten table is useful for matching each bus cycle with its status-line pattern.

#### till46 page 006

<img src="images/HandWrittenNotes/till46/page-006.jpg" width="720">

Explanation: This page explains the multiplexed low-order address/data bus. `AD0-AD7` first carry `A0-A7`, then later carry data, so `ALE` is used to latch the low address. The same page also gathers the serial pins `SID/SOD` and interrupt-related pins, so it is a combined pin-function recap.

#### till46 page 007

<img src="images/HandWrittenNotes/till46/page-007.jpg" width="720">

Explanation: This page introduces interrupt vector addresses and priority. It shows that restart-type interrupts use fixed vector addresses, such as multiplying the restart number by `8`. It also compares `TRAP`, `RST 7.5`, `RST 6.5`, `RST 5.5`, and `INTR` by priority and vector behavior.

#### till46 page 008

<img src="images/HandWrittenNotes/till46/page-008.jpg" width="720">

Explanation: This page compares memory-mapped I/O with I/O-mapped I/O. The notes connect `IO/M`, address width, port selection, and memory selection. The key distinction is whether the external device is treated like a memory location or as a separate I/O port.

#### till46 page 009

<img src="images/HandWrittenNotes/till46/page-009.jpg" width="720">

Explanation: This page ties `ALE`, machine cycle, and `T`-state together. It shows that the processor does not do a whole instruction in one step; it breaks work into machine cycles, and each machine cycle is made of clocked `T`-states. `ALE` appears early so the low address can be separated from the multiplexed bus.

#### till46 page 010

<img src="images/HandWrittenNotes/till46/page-010.jpg" width="720">

Explanation: This page separates instruction cycle, machine cycle, and `T`-state. An instruction cycle is the full execution of one instruction, machine cycles are bus operations inside it, and `T`-states are individual clock periods. The status-line examples show how opcode fetch, memory read/write, I/O read/write, and interrupt acknowledge are identified.

#### till46 page 011

<img src="images/HandWrittenNotes/till46/page-011.jpg" width="720">

Explanation: This page uses examples such as `MOV` and `ADD B` to show how instructions are executed through fetch, decode, and execute steps. It also reinforces that an instruction can require one or more machine cycles depending on whether only the opcode is needed or extra memory/data access is required.

#### till46 page 012

<img src="images/HandWrittenNotes/till46/page-012.jpg" width="720">

Explanation: This page defines basic instruction terms: word, instruction, opcode, operand, address bus, and data bus. It also explains that the 8085 has a 16-bit address bus and an 8-bit data bus, so it can select many memory locations but move one byte of data at a time.

#### till46 page 013

<img src="images/HandWrittenNotes/till46/page-013.jpg" width="720">

Explanation: This page draws the opcode-fetch timing diagram. Follow the signals in order: address appears, `ALE` latches the low address, `/RD` goes active, and the opcode is read on the data bus. The `MVI B,05H` example below it starts showing why the first byte is opcode and the next byte is immediate data.

#### till46 page 014

<img src="images/HandWrittenNotes/till46/page-014.jpg" width="720">

Explanation: This page continues `MVI B,05H`. The note shows that `06H` is the opcode for loading register `B`, and `05H` is the immediate data byte placed after it in memory. It also sketches an I/O read idea, where the port address and data transfer happen through a machine cycle.

#### till46 page 015
